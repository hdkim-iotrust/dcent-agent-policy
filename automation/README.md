# 주간 검수 자동 수집 (Google Apps Script · 자동 제안 + 사람 승인)

매주 금요일, 본인 구글 계정으로 실행되어:
1. 검수 추적 시트를 읽고
2. **매크로 처리 행 제외** (AI 답변(E)이 비어 있고 + 수정 근거(G)에 "매크로" 포함)
3. 실제 "수정"(F)이 들어간 행만 추려
4. 변경유형(RULE/FACT)을 가볍게 제안하고
5. 변경이력 시트에 **"검토 대기"**로 추가합니다. (이미 있는 건 중복 추가 안 함)

> policy.md 자동 수정/배포는 하지 않습니다. 금요일에 사람이 '검토 대기' 항목을 보고
> policy.md(이 repo)를 직접 수정 → Commit → 변경이력 시트의 적용여부를 '완료'로 바꿉니다.
>
> 서비스 계정·GitHub 시크릿 불필요. 본인 계정 권한으로 동작합니다.
> (이 .gs 파일은 기록용 정본입니다. 실제 실행은 아래처럼 Apps Script에 붙여넣어야 합니다.)

---

## 설정 (1회, 약 5분 · 비개발자용)

1. **변경이력 시트** 열기 → 상단 메뉴 **확장 프로그램 → Apps Script**
2. 편집기에 있는 기본 코드 지우고 → `automation/weekly_review.gs` 내용 **전체 붙여넣기** → 저장(💾)
3. 함수 목록에서 **weeklyReview** 선택 → **▶ 실행**
   - 처음 실행 시 권한 요청 → 본인(hdkim@iotrust.kr) 계정으로 **허용**
   - (변경이력 시트에 '검토 대기' 행이 추가되면 정상)
4. 좌측 **시계 아이콘(트리거)** → **트리거 추가**
   - 실행할 함수: `weeklyReview`
   - 이벤트 소스: **시간 기반**
   - 트리거 유형: **주 단위 타이머** → **금요일** → 오전 9~10시
   - 저장
5. 끝. 이후 매주 금요일 자동 실행됩니다.

> 참고: Apps Script가 두 시트에 접근하려면, 실행하는 본인 계정이 두 시트 모두에 접근 권한이 있어야 합니다(추적=보기 이상, 변경이력=편집).

---

## 매주 금요일 사람이 할 일 (5~10분)
1. 변경이력 시트에서 **'검토 대기'** 항목 확인
2. 진짜 반영할 것만: 변경유형(RULE/FACT) 확정 → policy.md에 반영(GitHub 웹 편집 → Commit)
   - FACT(정답·URL)는 가능하면 QnA 지식 시트에 반영
3. 변경이력 시트의 **적용여부**를 '완료'로, **agent 수정 내용**에 실제 반영 내용 기입
4. 반영하지 않기로 한 건은 적용여부 '보류/제외'로 표시

## 제외/포함 규칙
- 포함(학습 후보): "수정"(F) 칸에 내용이 있는 행 = AI 초안을 사람이 고친 건
- 제외:
  - "수정"(F) 칸이 비어 있는 행(초안 그대로 나감)
  - **"AI agent 답변 내용"(E)이 비어 있는 행** = 에이전트 초안이 없는 경우(매크로 처리, 문의 미비로 직접 답변 등) → 학습할 초안이 없으므로 제외
