# 주간 검수 자동 수집 (자동 제안 + 사람 승인)

매주 금요일(09:00 KST) GitHub Actions가 자동 실행되어:
1. 검수 추적 시트를 읽고
2. "기본 매크로 답변" 처리 행은 제외
3. 실제 "수정"이 들어간(=수정 필요) 행만 추려
4. 변경유형(RULE/FACT)을 가볍게 제안하고
5. 변경이력 시트에 **"검토 대기"**로 추가합니다.

> policy.md 자동 수정/배포는 하지 않습니다. 금요일에 사람이 '검토 대기' 항목을 보고
> policy.md(이 repo)를 직접 수정 → Commit → 변경이력 시트 적용여부를 '완료'로 바꿉니다.

---

## 최초 1회 설정 (비개발자용)

### 1) 구글 서비스 계정 만들기
1. https://console.cloud.google.com 접속 → 프로젝트 하나 생성(아무 이름)
2. 좌측 "API 및 서비스" → "라이브러리" → **Google Sheets API** 검색 → **사용 설정**
3. "API 및 서비스" → "사용자 인증 정보" → **사용자 인증 정보 만들기** → **서비스 계정**
   - 이름 아무거나(예: glean-weekly) → 만들기 → 완료
4. 만든 서비스 계정 클릭 → **키** 탭 → **키 추가** → **새 키 만들기** → **JSON** → 다운로드
   - 받은 JSON 파일 안에 `client_email`(예: glean-weekly@...iam.gserviceaccount.com)이 있습니다.

### 2) 두 시트를 서비스 계정에 공유
- **검수 추적 시트**: 위 `client_email`을 **뷰어**로 공유
- **변경이력 시트**: 위 `client_email`을 **편집자**로 공유

### 3) GitHub에 비밀키 등록
1. 이 repo → **Settings** → **Secrets and variables** → **Actions** → **New repository secret**
2. 이름: `GOOGLE_SA_KEY`
3. 값: 다운로드한 JSON 파일 **전체 내용**을 그대로 붙여넣기 → 저장

### 4) 동작 확인
- repo의 **Actions** 탭 → "Weekly Review Collector" → **Run workflow**(수동 실행)
- 변경이력 시트에 '검토 대기' 행이 추가되면 성공. 이후 매주 금요일 자동 실행됩니다.

---

## 매주 금요일 사람이 할 일 (5~10분)
1. 변경이력 시트에서 **'검토 대기'** 항목 확인
2. 진짜 반영할 것만: 변경유형(RULE/FACT) 확정 → policy.md에 반영(GitHub 웹 편집 → Commit)
   - FACT(정답·URL)는 가능하면 QnA 지식 시트에 반영
3. 변경이력 시트의 **적용여부**를 '완료'로, **agent 수정 내용**에 실제 반영 내용 기입
4. 반영하지 않기로 한 건은 적용여부 '보류/제외'로 표시

## 제외/포함 규칙
- 포함(학습 후보): "수정"(F) 칸에 내용이 있는 행 = AI 초안을 사람이 고친 건.
- 제외: 
  - "수정"(F) 칸이 비어 있는 행(초안 그대로 나감, 고칠 게 없음)
  - **매크로 처리 건** = "AI agent 답변 내용"(E)이 비어 있고 + "수정 근거 및 사유"(G)에 "매크로"가 포함된 행
- 매크로 판별 키워드는 `automation/weekly_review.py`의 `MACRO_KEYWORDS`에서 조정.
