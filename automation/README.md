## 추적 시트 열 매핑 (헤더 이름 기반 자동 매핑)
`weekly_review.gs`는 이제 **열 인덱스(COL_*)가 아니라 헤더 "이름"으로 열을 자동으로 찾습니다.**
→ 시트 열 순서가 또 바뀌어도 코드 수정 불필요. 헤더 "이름"이 바뀔 때만 `HEADER_MATCHERS`의 키워드를 수정하세요.
- 티켓번호 = 헤더에 "티켓" 포함 (2026-07-09 기준 A열)
- AI 답변 = 헤더에 "Ai agent 답변"/"aiagent" 포함 (G열)
- 수정 = 헤더가 정확히 "수정" (N열) — "수정 근거"·"보완"과 구분
- 수정 근거 = 헤더에 "수정 근거" 포함 (O열)
- 필수 열(위 4개) 중 하나라도 못 찾으면 **잘못 수집 방지를 위해 중단**하고 실행 로그에 남깁니다.
- **중복 방지 = 티켓 ID 기준**(2026-07-09~): 행 번호가 아니라 티켓 ID로 비교하므로, 추적 시트 행이 밀리거나 티켓이 숫자/URL/하이퍼링크 등 어떤 표기여도 같은 티켓은 한 번만 수집됩니다. 변경이력에 저장되는 티켓은 항상 전체 URL(클릭 가능)로 통일됩니다.
- (참고 2026-07-09 전체 열: A티켓번호 B답변날짜 C답변시간 D상태 E제목 F문의내용 G AI답변 H~M 6원칙 N수정 O수정근거 P보완 Q오답변)

## 설정 / 갱신 (비개발자용)

1. **변경이력 시트** 열기 → 상단 메뉴 **확장 프로그램 → Apps Script**
2. 편집기 코드를 `automation/weekly_review.gs` 최신 내용으로 **전체 교체** → 저장(💾)
3. 함수 목록에서 **dailyReview** 선택 → **▶ 실행**
   - 처음 실행 시 권한 요청 → 본인(hdkim@iotrust.kr) 계정으로 **허용**
   - (변경이력 시트에 '검토 대기' 행이 추가되면 정상)
4. 좌측 **시계 아이콘(트리거)** → 기존 트리거가 있으면 **편집**, 없으면 **트리거 추가**
   - 실행할 함수: `dailyReview`  (기존 weeklyReview 트리거는 편집으로 이 함수로 바꾸거나 삭제)
   - 이벤트 소스: **시간 기반**
   - 트리거 유형: **일 단위 타이머** → **오전 10시~11시 사이**
   - 저장
5. 끝. 이후 매일 오전 10시 자동 실행됩니다.

> 참고: 본인 계정이 두 시트 모두 접근 권한이 있어야 합니다(추적=보기 이상, 변경이력=편집).
> ※ 함수명을 dailyReview로 바꿨으므로, 기존 weeklyReview 트리거를 그대로 두면 "함수 없음" 오류가 납니다. 반드시 편집 또는 삭제하세요.

4. 반영하지 않기로 한 건은 적용여부 '보류/제외'로 표시
